# Krennic AI Agent Rules

Read `AGENTS.md` first, then `krennic/docs/ai-agent-runbook.md`.

Before every coding task, fetch upstream and start from the newest GitHub state
when it is safe to fast-forward. Keep Krennic running as a background service.
After code changes, validate, commit, rebase/pull, and push. Use pull requests
and the repository's protected-branch policy for merges.
